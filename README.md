# Amazon Automation Testing Project

A Selenium-based automated test suite for the Amazon India website ([amazon.in](https://www.amazon.in)), built with Java, TestNG, and Maven.

---

## Setup & Prerequisites

1. **Java 8+** installed and `JAVA_HOME` configured.
2. **Maven** installed and available on `PATH`.
3. **Google Chrome** browser installed.
4. **ChromeDriver** matching your Chrome version — Selenium 4.x manages this automatically via Selenium Manager (no manual download needed).

### Clone the repository

```bash
git clone <your-repo-url>
cd amazon
```

### Configure test data

Open `data_fetching/data_fetching.xlsx` and fill in the `username` sheet (row 0, cell 0) with a valid Amazon test account email. Update the file path in `utility_test.java` to match your local machine path if needed.

### Update screenshot paths

In `utility_test.java`, update the hardcoded screenshot destination paths (`/Users/nagaraj.pd/...`) to a valid path on your machine:

```java
File Dist = new File("Screenshots/Pass/" + date1 + ".png");
File dist = new File("Screenshots/Fail/" + date1 + ".png");
```

## Running the Tests

### Run all tests via Maven

```bash
mvn test
```

This executes the suite defined in `amazon_testing_testng.xml`, which runs all 15 test classes in parallel (5 threads).

### Run via TestNG XML directly (Eclipse/IntelliJ)

Right-click `amazon_testing_testng.xml` → **Run As → TestNG Suite**.

## Test Reports

After a test run, reports are generated in:

- `test-output/index.html` — Full TestNG HTML report
- `test-output/emailable-report.html` — Summary report suitable for email
- `target/surefire-reports/` — Surefire reports (generated by Maven)

Screenshots are saved to:
- `Screenshots/Pass/` — Screenshots of passed tests
- `Screenshots/Fail/` — Screenshots of failed tests

## Framework Design

- **Page Object Model (POM):** Each page of the Amazon website has a dedicated class under `src/main/java`, keeping test logic separate from locator/interaction logic.
- **Base Class:** `Basee_class` handles `@BeforeMethod` (launch Chrome, open amazon.in) and `@AfterMethod` (quit browser) for every test.
- **Listeners:** `utility_test` implements `ITestListener` to automatically capture screenshots on test pass/failure.
- **Retry Logic:** `Retry_logic` retries any failing test once before marking it as failed.
- **Data-Driven:** `utility_test` includes an Excel data-fetching utility using Apache POI for reading credentials from `data_fetching.xlsx`.
- **Parallel Execution:** The TestNG suite runs test classes in parallel with a thread count of 5.
